Templates let you create consistent notes with predefined structure and automatic variable substitution.

## Template Tools

### list-templates

List available templates.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `folder` | string | No | Templates folder (default: 'templates') |

**Example prompts:**
- "Show me available templates"
- "List templates in my custom-templates folder"

---

### get-template

Get a template and show its variables.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | Yes | Template name (with or without .md) |
| `folder` | string | No | Templates folder (default: 'templates') |

Shows the template content and lists all variables it uses.

**Example prompts:**
- "Show me the meeting template"
- "What variables does the project template use?"

---

### apply-template

Create a new note from a template.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `template` | string | Yes | Template name |
| `path` | string | Yes | Target path for the new note |
| `variables` | string | No | Variables as 'key1=value1,key2=value2' |
| `template_folder` | string | No | Templates folder (default: 'templates') |

**Example prompts:**
- "Create a new meeting note using the meeting template"
- "Apply the project template to projects/New Website.md"
- "Create a book note with title='The Great Gatsby' and author='F. Scott Fitzgerald'"

## Template Variables

### Built-in Variables

These are automatically available:

| Variable | Description | Example |
|----------|-------------|---------|
| `{{date}}` | Current date (YYYY-MM-DD) | 2024-03-15 |
| `{{time}}` | Current time (HH:MM) | 14:30 |
| `{{title}}` | Note title from path | My Note |
| `{{filename}}` | Filename without extension | My Note |

### Custom Variables

Define your own variables and pass them when applying:

```markdown
---
title: {{title}}
author: {{author}}
status: draft
---

# {{title}}

Written by: {{author}}
Created: {{date}}
```

Apply with: `variables='author=Jane Doe'`

## Best Practices

### Naming Conventions

- Use descriptive, lowercase names: `meeting.md`, `book-notes.md`
- Group related templates: `work/meeting.md`, `personal/journal.md`

### Variable Naming

- Use descriptive names: `{{project_name}}` not `{{pn}}`
- Document required variables in template comments
- Provide sensible defaults where possible

### Template Design

1. **Start with frontmatter** - Include common metadata fields
2. **Add structure** - Use headings for consistent organization
3. **Include prompts** - Add placeholder text to guide note creation
4. **Keep it flexible** - Don't over-structure; leave room for customization

## Advanced Usage

### Combining with Periodic Notes

Templates can be used with periodic notes:

> "Create today's daily note using the daily-review template"

### Batch Creation

> "Create project notes for Alpha, Beta, and Gamma using the project template"

The AI will call `apply-template` multiple times with different titles.
