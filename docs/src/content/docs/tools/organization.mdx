Organization tools help you structure your vault, create navigation aids, and refactor content.

## Maps of Content (MOC)

### generate-moc

Generate a Map of Content from a directory.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `directory` | string | No | Directory to generate MOC from (default: vault root) |
| `title` | string | No | Title for the MOC (default: directory name + ' MOC') |
| `output` | string | No | Output path (if not set, returns content) |
| `group_by` | string | No | Grouping: none, alpha, tag (default: none) |
| `recursive` | boolean | No | Include subdirectories (default: false) |

**Example prompts:**
- "Generate a MOC for my projects folder"
- "Create an alphabetically grouped MOC for all notes"
- "Build a MOC for the learning/ directory including subfolders"

---

### update-moc

Update an existing MOC with new notes.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Path to the MOC file |
| `directory` | string | No | Directory to scan for new notes |
| `recursive` | boolean | No | Include subdirectories (default: false) |

Adds links to new notes while preserving existing content and structure.

**Example prompts:**
- "Update my projects MOC with any new project notes"
- "Refresh the learning MOC to include recent notes"

---

### discover-mocs

Find notes tagged with #moc.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `directory` | string | No | Limit to specific directory |

**Example prompts:**
- "Find all Maps of Content in my vault"
- "Show MOCs in the knowledge/ folder"

## Indexes

### generate-index

Generate an alphabetical index of all notes.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `directory` | string | No | Directory to index (default: vault root) |
| `output` | string | No | Output path (if not set, returns content) |
| `title` | string | No | Title (default: 'Index') |
| `include_orphans` | boolean | No | Include notes with no links/tags (default: true) |

Creates A-Z grouped index with links to all notes.

**Example prompts:**
- "Generate an alphabetical index of my vault"
- "Create an index for the reference/ folder"

## Note Refactoring

### split-note

Split a note into multiple notes at heading boundaries.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Path to the note to split |
| `level` | number | No | Heading level to split at (1-6, default: 2) |
| `keep_original` | boolean | No | Keep original after splitting (default: false) |
| `output_dir` | string | No | Directory for new notes (default: same as original) |

**Example prompts:**
- "Split this long note into separate notes for each H2 section"
- "Break up my meeting notes by topic headings"
- "Split the documentation into individual pages at H1"

---

### merge-notes

Merge multiple notes into a single note.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `paths` | string | Yes | Comma-separated paths or JSON array |
| `output` | string | Yes | Output path for merged note |
| `separator` | string | No | Separator between content (default: horizontal rule) |
| `delete_originals` | boolean | No | Delete originals after merge (default: false) |
| `add_headings` | boolean | No | Add filename as heading if missing (default: true) |

**Example prompts:**
- "Merge these three related notes into one"
- "Combine all my meeting notes from this week into a summary"
- "Consolidate duplicate notes about Python"

---

### extract-section

Extract a heading section to a new note.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Path to the source note |
| `heading` | string | Yes | Heading text to extract |
| `output` | string | No | Output path (default: heading name) |
| `remove_from_original` | boolean | No | Remove section from original (default: true) |
| `add_link` | boolean | No | Add link to extracted note (default: true) |

**Example prompts:**
- "Extract the 'Installation' section to its own note"
- "Move the 'API Reference' heading to a separate file"
- "Pull out the 'History' section but keep a link"

---

### duplicate-note

Create a copy of a note.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Path to the note to duplicate |
| `output` | string | No | Output path (default: original + ' (copy)') |

**Example prompts:**
- "Duplicate this template note"
- "Create a copy of my project template as a new project"

## Vault Maintenance

### find-stubs

Find notes with few words that may need expansion.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `max_words` | number | No | Maximum word count to be a stub (default: 100) |
| `directory` | string | No | Limit to specific directory |
| `limit` | number | No | Maximum results (default: 50) |

**Example prompts:**
- "Find stub notes that need more content"
- "Show notes under 50 words"
- "Find underdeveloped notes in my concepts/ folder"

---

### find-outdated

Find notes not modified in a while.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `days` | number | No | Days since last modification (default: 90) |
| `directory` | string | No | Limit to specific directory |
| `limit` | number | No | Maximum results (default: 50) |

**Example prompts:**
- "Find notes I haven't touched in 6 months"
- "Show outdated notes in my projects folder"
- "What notes need review?"

---

### vault-stats

Get statistics about the vault.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `directory` | string | No | Limit stats to specific directory |

Returns: total notes, folders, words, links, tags, orphans, etc.

**Example prompts:**
- "Show me vault statistics"
- "How many notes do I have in projects/?"

## Workflows

### Content Reorganization

1. **Find candidates**: Use `find-stubs` and `find-outdated`
2. **Analyze structure**: Use `get-headings` to see organization
3. **Refactor**: Use `split-note`, `merge-notes`, or `extract-section`
4. **Update navigation**: Use `update-moc` to refresh links

### Creating a Knowledge Hub

1. **Generate MOC**: `generate-moc` for your topic folder
2. **Add to index**: `generate-index` for vault-wide navigation
3. **Link MOCs**: Create a "Home" note linking to MOCs
4. **Maintain**: Periodically run `update-moc` on each
