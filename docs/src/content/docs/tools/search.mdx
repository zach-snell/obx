Search tools help you find notes and content across your vault using various criteria.

## Basic Search

### search-vault

Search for content using case-insensitive substring matching.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | Search query |
| `directory` | string | No | Limit to specific directory |

**Example prompts:**
- "Search for 'machine learning' in my vault"
- "Find notes mentioning 'quarterly review' in projects/"

---

### search-by-tags

Search for notes by tags.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `tags` | string | Yes | Comma-separated tags (AND operation) |
| `directory` | string | No | Limit to specific directory |

**Example prompts:**
- "Find all notes tagged with #project"
- "Show notes with both #urgent and #work tags"
- "Search for #book-notes in my reading folder"

---

### search-headings

Search across all heading content in the vault.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | Search query for heading text |
| `level` | number | No | Filter by heading level (1-6, 0 for all) |
| `directory` | string | No | Limit to specific directory |

**Example prompts:**
- "Find sections titled 'Summary'"
- "Search for H2 headings containing 'API'"

## Advanced Search

### search-advanced

Advanced search with multiple terms and logical operators.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | Query with 'quoted phrases' and multiple terms |
| `in` | string | No | Where to search: content, frontmatter, tags, all (default: content) |
| `operator` | string | No | Logical operator: and, or (default: and) |
| `directory` | string | No | Limit to specific directory |
| `limit` | number | No | Maximum results (default: 50) |

**Query syntax:**
- Multiple terms: `python tutorial` (both must match)
- Exact phrases: `"machine learning"` (matches exact phrase)
- Mixed: `python "deep learning" tutorial`

**Example prompts:**
- "Find notes containing both 'React' and 'TypeScript'"
- "Search for exact phrase 'project management' in frontmatter"
- "Find notes with 'API' or 'REST' in the content"

---

### search-by-date

Find notes by modification or creation date range.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `from` | string | No | Start date (YYYY-MM-DD, MM-DD-YYYY, etc.) |
| `to` | string | No | End date |
| `type` | string | No | Date type: modified, created (default: modified) |
| `directory` | string | No | Limit to specific directory |
| `limit` | number | No | Maximum results (default: 50) |

**Supported date formats:**
- `2024-01-15` (YYYY-MM-DD)
- `01-15-2024` (MM-DD-YYYY)
- `Jan 15, 2024`

**Example prompts:**
- "Find notes modified in the last week"
- "Show notes created between Jan 1 and Mar 31"
- "What did I work on yesterday?"

---

### search-regex

Search using regular expression patterns.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `pattern` | string | Yes | Regular expression pattern |
| `case_insensitive` | boolean | No | Case insensitive (default: true) |
| `directory` | string | No | Limit to specific directory |
| `limit` | number | No | Maximum results (default: 50) |

**Example patterns:**
- `TODO:.*` - Find TODO comments
- `\d{4}-\d{2}-\d{2}` - Find dates in YYYY-MM-DD format
- `https?://[^\s]+` - Find URLs
- `\[\[.*\]\]` - Find wikilinks

**Example prompts:**
- "Find all notes containing URLs"
- "Search for ISBN patterns"
- "Find notes with TODO items"

## Search Tips

### Combining Searches

Ask the AI to combine multiple searches:

> "Find notes tagged #project that were modified this week and contain 'deadline'"

The AI will chain `search-by-tags`, `search-by-date`, and `search-vault` tools.

### Narrowing Results

Use directory filters to focus searches:

> "Search for 'API' only in my work/ folder"

### Understanding Results

Search results typically include:
- **Path**: Location of the matching note
- **Context**: Surrounding text showing the match
- **Line number**: Where the match was found
- **Match count**: How many times the term appears
