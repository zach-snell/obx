Canvas tools let you create and manipulate Obsidian's visual canvas files (`.canvas`).

## What is Canvas?

[Obsidian Canvas](https://obsidian.md/canvas) is a visual tool for organizing notes and ideas spatially. Canvas files contain:

- **Nodes**: Cards that hold text, links to notes, or web links
- **Edges**: Connections between nodes
- **Groups**: Visual groupings of nodes

## Canvas Tools

### list-canvases

List all canvas files in the vault.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `directory` | string | No | Limit to specific directory |

**Example prompts:**
- "Show all canvases in my vault"
- "List canvases in the projects folder"

---

### read-canvas

Read and parse a canvas file.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Path to the canvas file |

Returns the canvas structure including all nodes, edges, and their properties.

**Example prompts:**
- "Show me the contents of my project-map.canvas"
- "What nodes are in my brainstorm canvas?"

---

### create-canvas

Create a new empty canvas.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `path` | string | Yes | Path for the new canvas |

**Example prompts:**
- "Create a new canvas called project-overview.canvas"
- "Make a blank canvas in my planning folder"

---

### add-canvas-node

Add a node to a canvas.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `canvas` | string | Yes | Path to the canvas file |
| `type` | string | No | Node type: text, file, link, group (default: text) |
| `content` | string | No | Content (text/path/URL depending on type) |
| `x` | number | No | X position (default: 0) |
| `y` | number | No | Y position (default: 0) |
| `width` | number | No | Node width (default: 300) |
| `height` | number | No | Node height (default: 200) |

**Node types:**
- `text`: A card with markdown text
- `file`: A link to a note in your vault
- `link`: An embedded web link
- `group`: A visual group container

**Example prompts:**
- "Add a text node to my canvas with the title 'Main Idea'"
- "Add a link to my Project.md note on the canvas"
- "Create a group node labeled 'Phase 1' at position 100,100"

---

### add-canvas-edge

Add an edge between nodes in a canvas.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `canvas` | string | Yes | Path to the canvas file |
| `from` | string | Yes | Source node ID |
| `to` | string | Yes | Target node ID |
| `label` | string | No | Optional edge label |

**Example prompts:**
- "Connect node A to node B on my canvas"
- "Add an arrow from 'Start' to 'Process' with label 'begins'"

## Canvas File Format

Canvas files are JSON with this structure:

```json
{
  "nodes": [
    {
      "id": "abc123",
      "type": "text",
      "text": "My idea",
      "x": 0,
      "y": 0,
      "width": 300,
      "height": 200
    },
    {
      "id": "def456",
      "type": "file",
      "file": "notes/Project.md",
      "x": 400,
      "y": 0,
      "width": 300,
      "height": 200
    }
  ],
  "edges": [
    {
      "id": "edge1",
      "fromNode": "abc123",
      "toNode": "def456",
      "label": "relates to"
    }
  ]
}
```

## Use Cases

### Project Planning

> "Create a canvas for my new project with nodes for: Requirements, Design, Implementation, Testing"

The AI will:
1. Create a new canvas
2. Add four text nodes positioned in a flow
3. Optionally connect them with edges

### Mind Mapping

> "Build a mind map canvas with 'Machine Learning' in the center and branches for: Supervised, Unsupervised, Reinforcement"

### Linking Notes Visually

> "Create a canvas showing how my MOC notes connect to each other"

The AI will:
1. Create a canvas
2. Add file nodes for each MOC
3. Add edges based on wikilinks between them

### Meeting Planning

> "Make a canvas for the quarterly review with nodes for each department"

## Workflows

### Building a Canvas Incrementally

```
You: "Create a new canvas called 'roadmap'"
AI: Creates roadmap.canvas

You: "Add a node for Q1 goals"
AI: Adds text node, returns node ID

You: "Add a node for Q2 goals to the right of Q1"
AI: Adds text node at x+400

You: "Connect Q1 to Q2 with label 'leads to'"
AI: Adds edge between nodes
```

### Canvas from Search Results

> "Create a canvas with all my notes tagged #key-concept"

The AI will:
1. Search for notes with the tag
2. Create a canvas
3. Add file nodes for each matching note
4. Arrange them in a grid

## Tips

### Node Positioning

- Default canvas center is (0, 0)
- Positive X goes right, positive Y goes down
- Standard node width: 300px
- Leave ~100px gaps between nodes

### Working with Node IDs

When adding edges, you need node IDs. Ask the AI to:

> "Read my canvas and show me the node IDs"

Then reference them:

> "Connect node abc123 to node def456"

### Combining with Other Tools

- Use `search-by-tags` to find notes for the canvas
- Use `backlinks` to map note relationships
- Use `forward-links` to understand connections
