While the core logic of `obx` supports over 70 distinct actions, exposing all of those to modern LLMs (like Claude or GPT-4o) frequently causes the intelligent agent to breach its hard tool limits when run alongside other MCP servers.

To maximize stability and ensure your assistant can handle complex multi-server workflows, `obx` multiplexes these 70+ actions into **14 unified MCP Tools**.

When your AI assistant needs to do something, it calls one of these 14 parent tools and passes an `action` argument (e.g. `action: "read"` vs `action: "write"`).
